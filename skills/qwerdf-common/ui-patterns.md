# UI Patterns

本文件记录 Product Delivery 中常见页面模式。选择模式时先看产品类型、用户任务和信息密度，而不是先套视觉风格。

## 1. SaaS / CRM / 管理后台 / 操作台

适用：高频使用、需要扫描 / 比较 / 批量处理 / 配置规则的业务系统。

默认模式：

- 稳定左侧或顶部导航，当前模块位置清楚。
- 顶部摘要只放决策必要指标，不做营销 hero。
- 主体优先表格、筛选、列表、详情抽屉、批量操作和状态反馈。
- 组件密度中到高，留白服务可读性，不服务装饰。
- 页面操作固定在可预测位置，例如表格右上、详情页底部或抽屉 footer。

常见反模式：

- 大面积渐变首屏、宣传文案、装饰插画占用操作空间。
- 所有内容都放进孤立卡片，导致扫描路径被打断。
- 只展示指标卡，不提供筛选、明细、操作和异常恢复。

## 2. 数据看板

适用：指标监控、运营分析、实时状态、风险观测。

- 先定义用户要回答的问题，再选择图表。
- 同屏指标必须有口径、单位、时间范围和更新时间。
- 高风险状态要能下钻到明细或处理动作。
- 图表旁保留筛选条件、数据异常说明和空 / 错状态。

## 3. 表单 / 配置页

- 按用户理解的业务分组，不按数据库字段分组。
- 必填和校验规则靠近输入项，错误信息可定位。
- 保存、取消、重置、危险操作和权限不足状态明确。
- 多步骤配置必须说明进度、依赖、保存策略和失败恢复。

## 4. 表格 / 列表页

- 默认给出筛选、排序、分页 / 加载更多、空态和错误态。
- 表格列按“识别对象 -> 判断状态 -> 执行动作”的顺序组织。
- 批量操作要显示选中数量和影响范围。
- 移动端改为卡片列表或摘要行，不照搬桌面表格。

## 5. 详情页

- 首屏回答“这是什么、当前状态、用户能做什么”。
- 详情分区按任务组织：摘要、关键字段、关联记录、历史 / 日志、危险操作。
- 编辑和审计信息要区分，避免在同一视觉层级混杂。

## 6. 设置页

- 导航按配置域组织，避免把所有设置堆在一个长表单。
- 默认值、继承关系、权限和保存范围必须清晰。
- 影响全局行为的配置必须有确认、预览或回滚方式。

## 7. 移动端工具

- 单列优先，关键动作靠近拇指区域。
- 折叠次要筛选，保留搜索、状态和主要 CTA。
- 输入组件使用合适键盘类型，避免桌面 hover 作为唯一反馈。

## 8. Landing Page

适用：PRD 明确目标是获客、介绍、转化或活动页面。

- 首屏 headline 应是品牌、产品、对象或明确 offer，不用空泛价值口号替代。
- 支撑文案说明用户问题、核心能力和证据。
- 视觉资产必须展示真实产品、场景或对象；不使用纯装饰渐变当主资产。
- 页面必须露出下一段内容线索，避免首屏像单张海报。
