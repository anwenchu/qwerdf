# Visual Defects Reference

用于 `$pd-ui-review` 识别基础视觉缺陷。finding 必须绑定位置、证据、影响和修复建议。

## 高优先级缺陷

- 遮挡：固定头部、侧栏、底栏、toast、modal、drawer、popover 或 tooltip 覆盖主内容、表单字段、提交按钮或错误提示。
- 重叠：文本与图标、badge、按钮、表格列、卡片标题、图表标签或浮层互相压住。
- 溢出：长文本、长单词、数字、状态标签、表格列、筛选器、面包屑或按钮超出容器；移动端出现页面级横向滚动。
- 不可操作：按钮太小、点击目标太密、disabled / loading 不清楚、提交后无反馈、错误原因远离字段。
- 设计系统漂移：颜色、字体、圆角、阴影、边框、按钮层级、icon 风格明显偏离 `MASTER` 或 Page Overrides。

这些缺陷属于低级视觉问题零容忍项。若截图或 frame 中仍存在，不能写“已通过”；必须输出 P0/P1/P2 finding、归属和复查方式。

## 基础专业度缺陷

- 风格不统一：同页混用多个按钮风格、卡片样式、分隔线、阴影或图标集。
- 字体不一致：同层级的字号、字重、行高不同；表格、侧栏或紧凑面板使用 hero 级大字。
- 不整齐：左边缘错位、基线不齐、卡片高度混乱、按钮宽高不一、表单 label / input 对不齐。
- 间距混乱：相邻模块距离缺少规律，表单行距不一致，页面边距与组件间距脱离设计系统。
- 突兀：过亮色块、过重阴影、过多装饰、营销 hero 或低密度大卡片压过后台系统核心任务。
- 图标误用：用 emoji 当结构性图标；同页混用填充、线性、双色或不同 stroke width 的 icon。

## 复杂信息展示缺陷

- 信息优先级缺失：列表 / 表格 / 卡片 / 详情页没有区分主识别信息、状态、关键指标、次要属性和操作。
- 模板不一致：同类数据在不同卡片、行、详情块里使用不同对齐、分隔、格式或密度。
- 过度压缩：字段太多时只缩小字体或压缩间距，没有分组、摘要、展开、详情页或 tooltip。
- 移动端降级缺失：桌面表格直接压到 375px，导致页面级横向溢出或操作区挤压变形。
- 数值格式漂移：金额、百分比、时间、状态、枚举在同页面格式和对齐不一致。

## 状态缺陷

- 关键页面缺 default / loading / empty / error / permission / saving / submitting。
- empty 只表达情绪，不解释为什么为空或下一步动作。
- error 只写失败，不提供恢复路径；后端异常原文直接暴露给用户。
- destructive action 与普通操作视觉层级相同，缺少确认或后果说明。

## Finding 写法

```text
级别: P1
类别: 遮挡 / 响应式
区域: 任务创建页 mobile 375px 底部提交区
问题: fixed bottom action bar 覆盖最后两个必填字段和错误提示。
影响: 用户无法确认字段错误，移动端主流程无法提交。
建议: 为表单底部增加 safe-area padding，并让错误提示滚动到固定栏上方。
证据: screenshot mobile-375.png，底部 80px 区域。
归属: $pd-fe
```
