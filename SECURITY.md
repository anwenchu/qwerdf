# Security

Do not include tokens, cookies, private keys, customer data, or proprietary workspace artifacts in issues, pull requests, benchmark reports, or skill outputs.

## Reporting

Use GitHub Security Advisories if this repository is hosted on GitHub with private vulnerability reporting enabled.

Until a dedicated security email is published, use the safest maintainer-controlled channel available. If no private channel exists, open a minimal public issue titled `Security: private contact requested` and include only the affected area. Do not include secrets, customer data, exploit code, or step-by-step abuse details in public issues.

Security-sensitive fixes should include:

- affected skill or script
- impact
- reproduction steps using synthetic data
- validation performed after the fix
